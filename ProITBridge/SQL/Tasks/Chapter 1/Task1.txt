--------- SQL Task 1 ------------------

1. What is the use of the CREATE command in SQL?
CREATE is a DDL (Data Definition Language) command used to create database objects such as tables, databases, indexes, views, and schemas.
Ex : 
CREATE TABLE employees (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);

2. What is the difference between DROP and TRUNCATE?
| Feature   | DROP                  | TRUNCATE         |
| --------- | --------------------- | ---------------- |
| Action    | Deletes entire object | Deletes all rows |
| Structure | Removed               | Kept             |
| Rollback  | No (mostly)           | No (in many DBs) |
| Speed     | Slow                  | Faster           |
Ex:
DROP TABLE employees; -- deletes table completely
TRUNCATE TABLE employees; -- deletes all rows, keeps table

3. What does the ALTER command do in SQL?
ALTER is used to modify the structure of an existing database object, such as adding, deleting, or modifying columns.
EX: 
ALTER TABLE employees ADD salary INT;
ALTER TABLE employees MODIFY name VARCHAR(100);

4. What is the purpose of the SELECT statement?
SELECT is used to retrieve data from one or more tables, with support for filtering, aggregation, joins, and transformations.
EX : 
SELECT name, salary 
FROM employees 
WHERE salary > 50000;

5. What is a PRIMARY KEY and why is it important?
A PRIMARY KEY is a column (or set of columns) that:
* uniquely identifies each row
* cannot contain NULLs
* enforces entity integrity
EX : 
CREATE TABLE employees (
    id INT PRIMARY KEY
);

6. What is the difference between PRIMARY KEY and UNIQUE?
| Feature         | PRIMARY KEY  | UNIQUE              |
| --------------- | ------------ | ------------------- |
| NULLs           | Not allowed  | Allowed (usually 1) |
| Count per table | Only 1       | Multiple allowed    |
| Purpose         | Row identity | Enforce uniqueness  |
EX:
CREATE TABLE users (
    id INT PRIMARY KEY,
    email VARCHAR(100) UNIQUE
);
